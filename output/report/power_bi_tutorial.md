## 第 1 章：环境准备

### 1.1 下载与安装 Power BI Desktop

**Power BI Desktop 是微软官方免费提供的桌面版工具**，个人学习和制作看板完全免费。

#### 步骤：

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① | 打开浏览器，访问 `https://www.microsoft.com/zh-cn/download/details.aspx?id=58494` | 微软官网下载页面 |
| ② | 点击「下载」按钮 → 选择语言「中文(简体)」→ 点击下载 | 文件约 500MB |
| ③ | 运行下载的 `.msi` 安装包 | 双击即可启动安装向导 |
| ④ | 按照提示一路点击「下一步」「我接受」「安装」 | 默认选项即可 |
| ⑤ | 安装完成后点击「完成」，桌面上会出现 Power BI 图标 | 图标是黄色/橙色的 |

> 💡 **提示**：安装过程中如果提示需要 .NET Framework 或其他依赖，按提示同意安装即可。Power BI 只支持 Windows 系统。

### 1.2 首次启动与界面认识

双击桌面图标启动 Power BI Desktop，首次启动可能需要登录微软账户（可以用 Outlook / Hotmail 账号，没有也可以跳过）。

启动后你会看到如下界面布局（非常重要，后续每一步都会用到）：

```
┌─────────────────────────────────────────────────────────────┐
│  📌 顶部菜单栏：文件 / 开始 / 插入 / 视图 / 建模 / 帮助      │
├──────────┬────────────────────────┬──────────┬──────────────┤
│          │                        │          │              │
│  右侧    │     中间画布区         │   可视化  │   筛选器      │
│  字段列表 │  （拖拽图表到这里）     │   面板   │   面板        │
│          │                        │          │              │
│  显示所有 │                        │          │              │
│  可用字段 │                        │          │              │
│          │                        │          │              │
└──────────┴────────────────────────┴──────────┴──────────────┘
```

**五大区域说明：**

| 区域 | 位置 | 作用 |
|------|------|------|
| **顶部菜单栏** | 最上方 | 切换视图模式、数据建模、格式化等全局操作 |
| **字段列表** | 右上侧 | 显示当前数据表的所有列，**拖拽到图表里**的核心操作区 |
| **画布区** | 中央中间 | 放置和排列图表的主工作区 |
| **可视化面板** | 右中侧 | 选择图表类型（卡片/柱状图/饼图/表格等） |
| **筛选器面板** | 最右侧 | 为整个页面或单个图表添加筛选条件 |

> ⚠️ **新手最容易犯的错误**：在「可视化」面板选了图表后不知道下一步做什么。记住：**选好图表类型后，从右侧「字段列表」把字段拖进图表的字段框里**。

### 1.3 准备数据文件

我们的原始 CSV 有 **725MB / 约 900 万行**，直接导入 Power BI 会比较慢。因此提供了两种方案：

#### 方案 A：预聚合数据（推荐 ✅）

使用项目中自带的 Python 脚本一键导出聚合后的轻量数据集（总共只有几千行），导入秒开：

```bash
cd C:\Users\33062\PycharmProjects\PythonProject\taobao_user_analysis
python scripts/export_for_powerbi.py
```

运行成功后会生成以下文件到 `output/powerbi_data/` 目录：

| 文件名 | 行数 | 用途 |
|--------|------|------|
| `daily_metrics.csv` | ~31 行 | 日维度指标（PV/UV/购买量/转化率） |
| `hourly_metrics.csv` | 24 行 | 小时维度指标 |
| `behavior_summary.csv` | 4 行 | 行为类型汇总 |
| `conversion_funnel.csv` | 4 行 | 转化漏斗 UV 数据 |
| `rfm_segments.csv` | ~98 万行 | 用户级 RFM 分群标签 |
| `segment_summary.csv` | ~7 行 | 各分群统计汇总 |

#### 方案 B：直接导入原始 CSV

如果想挑战完整数据量，可以直接导入：
```
data/processed/user_behavior_cleaned.csv
```
⚠️ 导入时间约 2-5 分钟，且后续刷新也会较慢。建议先从方案 A 开始。

### 2.1 导入数据

以**方案 A（推荐）**为例，逐步导入 6 个 CSV 文件：

| ④ | 先选中 **`daily_metrics.csv`** → 点击「打开」 |
| ⑤ | 弹出「导航器」预览窗口 → 确认数据预览正确 → 点击右下角 **「加载」** |

重复步骤 ②–⑤，依次加载其余 5 个 CSV 文件：
```
hourly_metrics.csv
behavior_summary.csv
conversion_funnel.csv
rfm_segments.csv
segment_summary.csv
```

> 💡 **技巧**：也可以一次性选中多个 CSV 文件一起导入，Power BI 会自动识别每个文件为一个独立的「表」。

导入完成后，查看右侧 **「字段列表」**，你应该能看到 6 个表的名称：

```

📊 conversion_funnel
📊 rfm_segments
📊 segment_summary
```

### 2.2 检查与修正数据类型

Power BI 自动推断的数据类型不一定正确，需要手动检查和修正。

#### 操作步骤：

| 步骤 | 操作 |
|------|------|
| ① | 点击左侧边栏顶部的 **「数据视图」图标**（看起来像一个表格 📋） |
| ② | 在左侧表列表中点击 `daily_metrics` 表 |
| ③ | 逐列检查上方工具栏显示的数据类型： |

**daily_metrics 表 — 字段类型对照表：**

| 字段名 | 应设类型 | 设置方法 |
|--------|----------|----------|
| `date` | 🔵 **日期** | 选中该列 → 顶部工具栏「列工具」→ 类型下拉框改为「日期」 |
| `pv` | 🔢 **整数」 | 同上，改为「整数」 |
| `uv` | 🔢 **整数」 | 同上 |
| `cart_count` | 🔢 **整数」 | 同上 |
| `favorite_count` | 🔢 **整数」 | 同上 |
| `purchase_count` | 🔢 **整数」 | 同上 |
| `purchase_users` | 🔢 **整数」 | 同上 |
| `pv_to_purchase_rate` | 🔢 **定点小数」 | 同上 |
| `cart_to_purchase_rate` | 🔢 **定点小数」 | 同上 |

> ⚠️ **关键**：`date` 列**必须设为「日期」类型**，否则后续的时间序列图表无法正确排序和显示。

对 `hourly_metrics`、`conversion_funnel`、`behavior_summary` 等其他表也做同样的类型检查。

## 第 3 章：第一页 — 总览仪表盘

这是看板的**首页**，也是最重要的一页。目标是一眼让观看者了解整体业务状况。

### 3.1 创建 4 个 KPI 卡片

KPI 卡片（Card）是 Power BI 中最简单的可视化组件，适合展示单个关键数值。

#### 步骤：

| 操作 | 动作 |
|------|------|
| ① | 确保当前在 **「报表视图」**（左侧边栏第一个图标，像一页纸 📄） |
| ② | 在右侧 **「可视化」面板** 中，找到并点击 **「卡片」**（第一排的第一个，看起来像一个矩形卡片） |
| ③ | 画布中央出现一个空白卡片 → 从右侧 **「字段列表」** 展开 `daily_metrics` 表 |
| ④ | 将 **「浏览次数(PV)」** 字段拖入卡片的 **「字段」** 框（或直接勾选该字段前的复选框） |
| ⑤ | 卡片上会显示一个巨大的数字（如 `8,497,256`）|

重复以上步骤，再创建 3 个卡片，分别放入：

| 卡片序号 | 放入字段 | 显示效果 |
|----------|----------|----------|
| 卡片 1 | `浏览次数(PV)` | `8,497,256` |
| 卡片 2 | `独立访客(UV)` | `987,854` |
| 卡片 3 | `购买次数` | `99,486` |
| 卡片 4 | `浏览购买转化率(%)` | `1.17` |

#### 排列美化 4 个卡片：

| 操作 | 动作 |
|------|------|
| ① | 选中一个卡片 → 拖动边缘调整大小，使 4 个卡片等大 |
| ② | 把 4 个卡片横向排列成一排（可以手动对齐，或用 Ctrl+A 全选后用顶部「格式」→「对齐」） |
| ③ | 选中每个卡片 → 点击右上角的 **「格式化视觉对象」**（画笔图标 🖌️） |
| ④ | 展开下方的 **「数据标签」** → 设置字号为 **24**（加粗）、颜色为深色 |
| ⑤ | 展开 **「标题」** → 关闭标题（因为 KPI 卡片的字段名本身已足够清晰） |

最终效果预期：

```
┌──────────────┬──────────────┬──────────────┬──────────────┐
│              │              │              │              │
│   8,497,256  │    987,854   │    99,486    │     1.17%    │
│ 浏览次数(PV) │ 独立访客(UV) │   购买次数   │ 浏览购买转化率 │
│              │              │              │              │
└──────────────┴──────────────┴──────────────┴──────────────┘
```

### 3.2 制作转化漏斗图

漏斗图能直观展示从浏览到购买的流失情况。

#### 步骤：

| 操作 | 动作 |
|------|------|
| ① | 在可视化面板中找到 **「漏斗图」**（在「所有可视化」展开列表中搜索"漏斗"）并点击 |
| ② | 从 `conversion_funnel` 表中拖入字段： |

**漏斗图的字段配置：**

| 字段框 | 放入字段 | 说明 |
|--------|----------|------|
| **组（Group）** | `转化阶段` | 漏斗的每一层级名称 |
| **值（Values）** | `UV人数` | 每层的数值大小 |

> ⚠️ **注意**：Power BI 的原生漏斗图只支持单值递减。如果你发现漏斗图不可选或报错，改用下面的替代方案。

#### 替代方案：簇状条形图（更灵活）

如果漏斗图效果不理想，用条形图也能很好表达转化逻辑：

| 操作 | 动作 |
|------|------|
| ① | 选择 **「簇状条形图」**（横向柱状图） |
| ② | Y 轴放入 `转化阶段`，X 轴放入 `UV人数` |
| ③ | 排序：点击 Y 轴上的阶段名称 → 右键 → 「排序方式」→ 按 UV人数 降序 |
| ④ | 格式化：点击画笔图标 → 数据标签开启 → 显示「全部详细信息」（同时显示数值和百分比） |

| **值（Values）** | `行为占比(%)` | 每种行为的占比 |

#### 美化环形图：

| 操作 | 效果 |

## 第 6 章：DAX 度量值进阶

DAX（Data Analysis Expressions）是 Power BI 的公式语言。掌握基础 DAX 可以让你的看板从"静态报表"升级为"动态分析工具"。

### 6.1 什么是度量值？

**度量值（Measure）** 是基于 DAX 公式动态计算的值，它不会占用存储空间，每次查询时实时计算。

| 概念 | 类比 |
|------|------|
| **计算列** | 像 Excel 里新增一列，每个单元格存一个值，占存储空间 |
| **度量值** | 像一个公式，不存数据，用时现算，灵活强大 |

> 💡 **最佳实践**：优先使用度量值而非计算列。

### 6.2 新建度量值面板操作

| 操作 | 步骤 |
|------|------|
| ① | 在 **「报表视图」** 中，找到右侧 **「字段列表」** |
| ② | 选中你要添加度量值的表（比如 `daily_metrics`） |
| ③ | **右键点击**表名 → 选择 **「新建度量值」**（New Measure） |
| ④ | 上方出现公式编辑栏 → 输入 DAX 公式 → 按 Enter 保存 |
| ⑤ | 新建后的度量值会出现在该表下方，字段名前有一个计算器图标 🧮 |

### 6.3 核心度量值代码清单

以下是本项目最常用的 6 个度量值，**可直接复制粘贴使用**。

#### ① 总 PV

```dax
总 PV = COUNTROWS('daily_metrics')
```

> 说明：计算 `daily_metrics` 表的总行数（每天一行，所以就是天数）。如果要基于原始数据算真实总 PV，见下面注释版本。
>
> ```dax
> -- 如果用的是原始 user_behavior 表，用这版：
> 总 PV = COUNTROWS(FILTER('user_behavior', 'user_behavior'[behavior_type] = 1))
> ```

#### ② 总 UV（去重）

```dax
总 UV = SUM('daily_metrics'[独立访客(UV)])
```

#### ③ 浏览→购买转化率（%）

```dax
浏览购买转化率 % =
DIVIDE(
    SUM('daily_metrics'[购买次数]),
    SUM('daily_metrics'[浏览次数(PV)])
) * 100
```

> **DIVIDE 函数**：安全除法。当除数为 0 时返回空值而不是报错，比直接写 `/` 更稳健。

#### ④ 加购→购买转化率（%）

```dax
加购购买转化率 % =
DIVIDE(
    SUM('daily_metrics'[购买次数]),
    SUM('daily_metrics'[加购次数])
) * 100
```

#### ⑤ 复购率（%）

```dax
复购率 % =
VAR TotalBuyers = DISTINCTCOUNT('rfm_segments'[用户ID])
VAR RepeatBuyers = COUNTROWS(
    FILTER('rfm_segments', 'rfm_segments'[F-购买频次] >= 2)
)
RETURN DIVIDE(RepeatBuyers, TotalBuyers) * 100
```

> **解释**：`VAR` 定义临时变量，`RETURN` 返回结果。先算有购买记录的用户总数，再算其中购买 ≥2 次的用户数，相除得到复购率。

#### ⑥ 日均 PV

```dax
日均 PV =
AVERAGEX(
    VALUES('daily_metrics'[日期]),
    CALCULATE(SUM('daily_metrics'[浏览次数(PV)]))
)
```

> **解释**：`AVERAGEX` 逐行遍历每个日期，计算每天的 PV 再取平均值。

### 6.4 DAX 常用函数速查

| 函数 | 作用 | 示例场景 |
|------|------|----------|
| `SUM()` / `COUNTROWS()` / `AVERAGE()` | 基础聚合 | 求和/计数/求均值 |
| `DISTINCTCOUNT()` | 去重计数 | 计算独立访客数 UV |
| `DIVIDE(A, B)` | 安全除法 | 计算转化率（避免除零错误） |
| `CALCULATE(expr, filter)` | 改变筛选上下文 | 在特定条件下计算指标 |
| `FILTER(table, condition)` | 过滤行 | 筛选特定条件的记录 |
| `ALL(table/column)` | 忽略筛选器 | 计算总计/占比时用到 |
| `VALUES(column)` | 获取唯一值列表 | 用于 AVERAGEX 遍历 |
| `IF(condition, A, B)` | 条件判断 | 分类标注 |
| `RELATED(table[column])` | 跨表取值 | 多表关联时引用其他表字段 |
| `VAR ... RETURN` | 定义变量 | 复杂公式的可读性优化 |

## 附录

### 附录 A：常见问题排查

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| **导入 CSV 后中文乱码** | 编码问题（Power BI 默认用 ANSI 读 UTF-8） | 导入时在「文本/CSV」连接器中点击「转换数据」→ 顶部「源」步骤 → 修改编码为 **65001 (UTF-8)** |
| **日期字段显示为文本无法做时间轴** | date 列未被识别为日期类型 | 数据视图中选中该列 → 列工具 → 类型改为「日期」 |
| **图表拖入字段后不显示任何东西** | 字段被意外放到「工具提示」或「筛选器」而非正确的字段框 | 检查可视化面板右侧的各个字段框，确保字段在正确位置 |
| **折线图 X 轴顺序不对** | 日期被当作文本排序 | 确保 date 列是「日期」类型；如果是文本则转为日期 |
| **Power BI 启动很慢/卡死** | 内存不足或数据太大 | 关闭其他程序；或使用方案 A 的聚合数据（几百行 vs 几百万行） |
| **发布时报错「超出行数限制」** | 免费版 Power BI Service 单表上限 100 万行 | 使用聚合后的数据发布；或只发布不含 rfm_segments 大表的页面 |
| **找不到某个图表类型** | 可能在「更多图表」里 | 点击可视化面板底部的 **「...」(更多)** → 搜索图表名称 |
| **DAX 公式报语法错误** | 中文括号/引号问题 | 确保所有括号 `()` 和引号 `''` 都是英文半角字符 |

### 附录 D：面试问答模板

#### Q1：「你为什么选择用 Power BI 做这个看板？」

> **参考回答**：
> "首先 Power BI 是目前企业中使用最广泛的 BI 工具之一，学会它对实习工作有帮助。
> 其次它与我的 Python 技术栈衔接良好——我在 Python 中完成了数据清洗和 RFM 计算，
> 然后将聚合后的数据导出到 Power BI 做可视化，两者互补。
> 最后 Power BI 的 DAX 度量值让我能够实现动态指标计算，比静态的 Matplotlib 图更适合做交互式展示。"

#### Q2：「这个看板的最大亮点是什么？」

> **参考回答**：
> "我认为最大的亮点是**将 RFM 分群结果可视化**。
> 我不只是做了一个漂亮的图表，而是通过 RFM 模型把近 100 万用户分成 7 类群体，
> 发现了几个关键业务洞察：比如'加购→购买转化率高达 44%'，
> 这意味着运营应该重点关注如何把更多用户从浏览引导到加购环节。
> 这个结论是有数据支撑的可执行建议，而不只是数字堆砌。"

#### Q3：「你遇到了什么困难？怎么解决的？」

> **参考回答**：
> "主要难点是**数据处理量大**——原始数据有 900 万行、725MB。
> 我的解决方案是分层处理：
> 1. 用 Pandas 在 Python 中完成重度计算（RFM 分箱、分组聚合）
> 2. 将聚合后的数据导出为轻量 CSV（总共几千行）
> 3. 再导入 Power BI 做可视化
> 这样既保证了分析的准确性，又避免了 Power BI 直接处理大数据的性能瓶颈。"

#### Q4：「如果让你给运营团队提建议，你会说什么？」

> **参考回答**：
> "基于分析结果，我会提三点建议：
> 1. **重点抓加购环节**：数据显示加购用户的购买转化率是 44%，而浏览用户的购买率只有 1.17%，
>    差距 38 倍。所以运营策略应该是想尽办法让用户加入购物车——比如限时优惠提醒、满减凑单提示。
> 2. **针对流失用户做召回**：有购买记录但最近 7 天未活跃的用户约占 XX%，
>    可以通过优惠券/新品推送进行召回。
> 3. **利用晚间高峰时段**：晚上 20-22 点是行为高峰期，
>    大促活动、推送通知应集中在这个时间段投放。"
